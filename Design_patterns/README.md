# Design Patterns — A Concept-First Guide with Modern C++

A book-style study reference with a **two-layer approach**:

1. **Language-agnostic conceptual understanding** (primary) — *why* each pattern exists and how to reason about it.
2. **Modern C++ (C++14/17) implementation insights** (secondary) — memory, object lifetime, smart pointers, RAII, copy/move semantics, templates, and virtual dispatch.

## How to Read

- New to patterns? Read **Chapter 1** first — patterns make little sense without OOP + SOLID.
- Each pattern is taught **concept-first** (definition, problem, analogy, structure, working, tradeoffs, when/when-not) **before** the C++ code.
- Don't skip the **"Bad vs Improved"** C++ sections — that's where the real learning is.
- Use **Chapters 5 & 9** for revision and interviews.
